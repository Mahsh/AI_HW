# 作业三：内容安全审核项目——周报自动化与上下文窗口管理


## 一、业务背景

你是内容安全审核团队的一员。团队每周要处理大量审核工单，每个人在工单系统、企业微信、邮件和个人笔记中留下了碎片化的工作记录。

每到周五，你都需要把这些碎片整理成一份结构化周报——关键数据、本周完成、进行中的事项、风险和下周计划。但问题来了：

1. **工作记录很长**（50 多条碎片，加起来三四千字），AI 能消化得了吗？
2. 直接一股脑扔给 AI"帮我写周报"，AI 可能遗漏或搞混重要信息
3. 这份周报是每周都要写的——能不能做成一个模板，以后换个数据就能直接用？

本作业让你完整经历"生成 → 发现上下文限制 → 设计管理策略 → 做成模板"的全过程。

---

## 二、你将完成什么

整个作业分为 **4 个阶段**：

| 阶段 | 你在做什么 | 核心挑战 |
|------|-----------|---------|
| 阶段一 | 用分步骤推理的方法，让 AI 从碎片记录生成结构化周报 | 如何让 AI 不遗漏信息？ |
| 阶段二 | 发现上下文窗口的"遗忘"现象，尝试两种不同的上下文管理策略 | 长文本怎么喂给 AI？ |
| 阶段三 | 把最好的提示词做成可复用的模板 | 哪些部分应该变成变量？ |
| 阶段四 | 用 3 份不同的测试数据批量测试，迭代优化 | 模板是否真的通用？ |

## 四、分阶段任务说明

### 阶段一：用分步推理生成结构化周报

**目标**：不直接让 AI"帮我写周报"，而是设计分步骤的指令，让 AI 像人一样"先归纳、再撰写"。

#### 你的任务

1. **设计分步指令**：在你的提示词中，要求 AI 按照以下步骤处理（不要跳过步骤，每一步都要显式输出结果）：
   - **步骤1**：通读所有工作记录，按主题分类（审核工单 / 会议纪要 / 策略更新 / 异常事件 / 其他）
   - **步骤2**：归纳每个分类的关键产出和关键数据
   - **步骤3**：识别需要跨周关注的风险项（即"本周未解决、下周需要继续跟进"的事）
   - **步骤4**：按指定格式生成最终周报

2. **指定输出格式**：要求 AI 最终输出一份结构化的周报，包含以下板块：
   - 📌 本周概要（≤100字）
   - 📊 关键数据（审核量 / 通过率 / 异常数 / 响应时效等）
   - ✅ 本周完成（按主题分类，每条 ≤30字）
   - 🔄 进行中（标注完成度和阻塞点）
   - ⚠️ 风险与问题
   - 📅 下周计划

3. **开启新对话**，上传附件 `hw3_work_logs.md`，提交你的提示词。保存 AI 的完整回复。

```
├── response.md                  # AI 生成的周报

└── notes.md                     # 分步推理让周报质量提升了吗？具体体现在哪？
```

### 阶段二：上下文窗口管理实验

**目标**：发现在长文本场景下 AI 可能"遗忘"早期信息的问题，并尝试两种不同的解决方案。

#### 步骤 1：测试 AI 的"记忆"

在阶段一的同一个对话中（继续对话，不要开新的），追问 AI：
> "请告诉我本周第5条审核工单（CS-20260610-005）的具体内容是什么？"

- 如果 AI 能准确回答 → 说明你的工作记录长度还在 AI 的有效记忆范围内
- 如果 AI 记不清或胡编 → 说明出现了"上下文遗忘"现象
- 无论结果如何，**记录 AI 的回复**。

#### 步骤 2：尝试两种上下文管理策略

**策略 A：摘要压缩法**

1. **开一个全新的对话**
2. 先上传附件，对 AI 说："请将这份工作记录压缩为 300 字以内的摘要，保留所有关键数据和重要事件。"
3. 保存 AI 生成的摘要
4. **再开一个新对话**，将摘要 + 少量原始记录中最关键的部分（你自己挑选约 300 字）粘贴给 AI，要求生成周报
5. 保存最终周报

**策略 B：智能切分法**

1. 将附件内容按主题手动切成 3 段（例如：审核工单段 / 会议+策略段 / 异常+TODO段）
2. **开一个新对话**，依次让 AI 提取每段的关键信息：
   - 第 1 段：提取所有审核工单的关键数据（总数、通过率、拒绝类型分布）
   - 第 2 段：提取会议纪要和策略更新的要点
   - 第 3 段：提取异常事件和待办事项
3. 最后，**在同一对话中**，让 AI 将前面三轮提取的结果汇总成一份完整周报
4. 保存每一步的提示词和输出

#### 步骤 3：三种方案对比

从以下三个角度对比"原始一次输入""摘要压缩""智能切分"三种方案：
- **周报完整性**：哪个方案遗漏的信息最少？
- **关键数据准确率**：关键数字（审核量、通过率等）三个方案是否一致？
- **你估算的输入字数（Token 消耗）**：哪个最省？哪个最费？

### 阶段三：提示词模板化

**目标**：把最好的提示词变成模板，让团队成员换个数据就能直接用。

#### 你的任务

1. **识别变量**：回顾阶段一和阶段二中你效果最好的提示词版本，找出其中"每次都会变"的部分。至少定义 5 个变量，例如：
   - 报告人姓名
   - 报告周期（本周日期范围）
   - 工作记录内容（这是最大的变量——附件内容）
   - 重点关注的领域
   - 上周遗留的风险项

2. **编写模板**：将原始提示词中的固定内容保留，可变内容用 `{{变量名}}` 替换。模板需要包含：
   - 每个变量的说明（类型、是否必填、示例值）
   - 模板的适用场景说明

3. **测试模板**：将变量替换为一组"虚拟同事"的数据（不需要真的写一份新的工作记录，只需在变量说明中写出替换后的值），确认模板中哪些变量设计合理、哪些需要调整。

4. **缺失分析**：列出至少 3 个场景——如果某个变量忘了填或填错了，会导致什么问题？

#### 提交物

```
stage3_template_variables/
├── weekly_report_template.md    # 变量化后的 Prompt 模板 + 变量说明表
├── test_render.md               # 用另一组变量值渲染的测试结果
├── missing_detection.md         # 变量缺失/错填场景分析（≥3个场景）
└── notes.md
```

### 阶段四：批量测试 + 迭代优化

**目标**：用多份不同的测试数据验证你的模板，找到问题并改一轮。

#### 你的任务

1. **构造 3 份测试数据**：自己编写 3 份简短的"周工作记录"，每份 10-15 条记录，模拟三种不同的工作状态：
   - **测试1**：正常周——审核量平稳，无异常事件
   - **测试2**：故障多发周——频繁出现审核服务异常、误判事件
   - **测试3**：假期前一周——审核量减少，但有一堆行政事务和交接事项

2. **批量运行**：用阶段三的模板分别处理这 3 份测试数据，记录 3 份输出。

3. **自评打分**：为你自己的模板设定 3 个评估维度（如"数据准确度""结构完整性""语言流畅度"），对 3 份输出各打 1-5 分。

4. **迭代一轮**：根据评分最低的那个维度，修改你的模板中的对应部分。重新运行评分最低的那份测试数据，对比修改前后。

5. **最终交付**：输出一版你愿意"交给团队真实使用"的周报生成模板。

#### 提交物

```

├── testset/                     # 3 份测试数据
│   ├── test1_normal_week.md
│   ├── test2_incident_week.md
│   └── test3_holiday_week.md
├── batch_results.md             # 3 份输出 + 评分表
├── iteration_log.md             # 迭代记录（改了什么、为什么、改后对比）
├── final_template.md            # 最终交付版 Prompt 模板（含使用说明）
└── README.md                    # 作业三总说明
```
