阶段三安全加固心得体会
通过本次阶段三的提示词安全加固与二次攻击测试，我系统掌握了 AI 提示词纵深防御的完整设计思路，理解了 prompt 安全加固的核心逻辑：不是靠模型自带护栏，而是靠人为构建规则边界，从源头封杀注入攻击。
首先，我认识到原始提示词虽然可以勉强防御基础攻击，但属于“被动防御”，没有明确的安全规则，一旦遇到复杂变形注入、多轮嵌套越狱，很容易被绕过。因此必须主动增设结构化防护体系。
本次新增的四项防护形成了完整的四层防御体系：
第一，分隔符隔离解决了指令污染与层级覆盖问题，彻底杜绝用户输入篡改系统指令，是所有防护的基础。
第二，安全约束声明明确了系统规则优先级永久高于用户输入，从权限层面禁止用户越权改规则。
第三，输出范围白名单限制，实现了业务最小权限原则，超出业务范围的提问直接拒绝，避免被诱导输出敏感内容。
第四，异常检测机制实现了主动攻击识别，对覆盖指令、隐藏规则、角色扮演越狱三类高频攻击精准拦截。
经过三轮重测可以发现，加固后的提示词实现了全覆盖防御，所有攻击全部失效，模型行为高度可控、安全、稳定。
本次实验让我深刻理解了 Prompt 安全的核心思想：边界锁定、权限固化、输入隔离、异常拦截。未来在编写生产级提示词时，我会将这四项加固措施作为标准化模板，默认写入所有业务提示词，从源头抵御 Prompt 注入、越狱、越权问答等安全风险。
