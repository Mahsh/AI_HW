# 阶段三模板化设计笔记
## 一、变量筛选依据
对比阶段一一次性全量输入、阶段二智能切分两种方案，选取效果最优的「智能切分分步提取」作为模板底层逻辑；从中拆分出**6个可变变量**，区分必填/选填：
1. 固定不变逻辑：分步4步提取流程、周报输出格式、分段规则，全部固化在模板内，无需重复编写；
2. 高频变动内容：周期、汇报人、原始日志、本周业务重点，每周均不同，设为必填变量；
3. 低频变动内容：上周遗留风险、文字长度限制，按需填写，设为选填变量。

## 二、变量设计合理性总结
1. {{raw_work_logs}}：核心变量，承载全部业务数据，是周报生成基础，必填设计合理；
2. {{report_cycle}}：统一周报时间口径，避免多人汇报日期格式混乱；
3. {{focus_business}}：强制对齐团队当期工作重心，解决周报内容泛化无重点问题；
4. {{last_week_risk}}：实现风险闭环跟踪，承接每周历史问题，适合长期迭代使用。

## 三、模板复用优势
1. 团队全员统一提示词标准，不用每人重新编写分步指令；
2. 仅替换变量即可生成周报，降低重复写提示词的人力成本；
3. 底层采用智能切分上下文策略，规避长文本遗忘问题，周报数据准确率稳定；
4. 变量区分必填/选填，容错性高，少量选填项漏填不会完全失效。

## 四、可优化方向
1. 增加{{team_size}}团队人数变量，用于人均审核量统计；
2. 增加{{system_exception_threshold}}系统异常阈值变量，标准化时效风险判定；
3. 增加输出格式切换变量，支持「内部详细版」「管理层极简版」两种周报风格。
