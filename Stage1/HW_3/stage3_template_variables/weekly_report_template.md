# 内容安全审核团队周报生成通用提示词模板
## 模板适用场景
适用于每周内容安全审核碎片化工作记录自动生成结构化周报，支持长文本上下文智能切分提取（阶段二最优方案），兼顾数据完整性与低上下文负载，用于审核组内部业务复盘周报。

## 变量定义说明表
| 变量标识 | 变量名称 | 数据类型 | 是否必填 | 填写说明 & 示例 |
| ---- | ---- | ---- | ---- | ---- |
| {{report_cycle}} | 报告周期 | 文本 | 必填 | 本周时间范围，示例：2026年6月第3周（6.10-6.13） |
| {{reporter_name}} | 报告人 | 文本 | 必填 | 填写审核员/组长姓名，示例：张组长 |
| {{raw_work_logs}} | 原始工作记录全文 | 长文本 | 必填 | 工单、会议、策略、异常、待办完整碎片记录 |
| {{focus_business}} | 本周重点关注业务 | 文本数组 | 必填 | 当前核心业务方向，示例：["5G消息群聊治理","变形敏感词库优化","等保三级年审自查"] |
| {{last_week_risk}} | 上周遗留风险事项 | 文本 | 选填 | 上周未闭环风险，无则填“无”，示例：隐晦色情AI识别准确率不足 |
| {{output_limit}} | 单条事项字数限制 | 数字 | 选填 | 默认30，控制每条完成事项文字长度 |

## 标准化分步执行指令（固定逻辑，不可修改）
请严格执行4步流程，每一步输出分段结果，禁止跳步：
步骤1：对{{raw_work_logs}}按业务模块智能切分3段：
分段1：全部审核工单数据；分段2：会议纪要+策略更新；分段3：异常事件+个人待办TODO
分段2：逐段提取核心信息，分别输出三段结构化提取结果；
分段3：结合{{last_week_risk}}识别跨周延续风险、未闭环待办；
步骤4：整合三段提取数据，按照固定周报格式输出完整周报，严格贴合{{focus_business}}重点业务展开。

## 固定输出周报格式（不可修改）
# 内容安全审核团队周报【{{report_cycle}}】
汇报人：{{reporter_name}}
📌 本周概要（≤100字）
📊 关键数据（审核总量、通过率、拒绝分布、系统时效、复审量）
✅ 本周完成（按工单/策略/会议/异常分类，每条≤{{output_limit}}字）
🔄 进行中事项（标注完成度、截止时间）
⚠️ 风险与问题（含上周遗留风险{{last_week_risk}}延续情况）
📅 下周计划（匹配{{focus_business}}重点业务）
