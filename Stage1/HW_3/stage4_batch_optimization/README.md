# 阶段四：批量测试+模板迭代优化作业说明
## 一、实验目标
1. 构造3类差异化业务场景测试集，验证模板通用性；
2. 通过三维度量化打分定位模板短板；
3. 针对性迭代优化模板逻辑，产出可落地团队通用提示词模板。

## 二、3套测试数据集设计思路
1. test1_normal_week：常规平稳业务周期，无线上故障，核心为内容识别规则迭代，用于验证基础数据提取准确性；
2. test2_incident_week：多起连续线上服务宕机、AI误判舆情，高风险场景，暴露原模板多故障合并描述的缺陷；
3. test3_holiday_week：工单流量下降，大量人员交接、行政考核事务，验证模板对非技术类工作的承载能力。

## 三、评估体系说明
设立3项核心评估维度，1-5分打分，覆盖数据、结构、业务落地三层需求：
1. 数据准确度：量化数字、工单、故障、待办无错漏；
2. 结构完整性：六大固定板块齐全，事项边界清晰；
3. 业务贴合度：周报行文、篇幅、计划匹配本周业务特征。

## 四、迭代优化总结
原始模板短板：未区分多起独立异常事件，故障场景输出笼统；无周类型权重控制，不同业务周期周报侧重点同质化。
优化方案：新增故障逐条拆分强制约束、周类型变量、风险三类分类规则，优化后高故障场景评分从3.7提升至5分，模板适配全业务场景。

## 五、交付模板使用说明
1. 团队成员仅需替换7个变量即可一键生成周报，无需重复编写分步指令；
2. 内置智能切分上下文策略，解决长工作记录AI遗忘问题；
3. 自动区分正常/故障/假期三类工作周期，周报重点自动倾斜；
4. 强制拆分多起线上故障，风险分层展示，满足审核团队复盘、运维同步需求。
