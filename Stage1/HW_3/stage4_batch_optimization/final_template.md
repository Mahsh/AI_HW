# 内容安全审核团队通用周报生成模板（阶段四迭代优化终版）
## 适用场景
内容安全审核组每周碎片化工作记录自动生成结构化业务周报；兼容平稳周、故障多发周、假期交接周三类场景，采用智能切分上下文策略，规避长文本遗忘，支持团队全员复用。

## 变量定义说明表
| 变量标识 | 变量名称 | 类型 | 必填 | 填写示例 |
| ---- | ---- | ---- | ---- | ---- |
| {{report_cycle}} | 报告周期 | 文本 | 必填 | 2026年6月第3周（6.16-6.19） |
| {{reporter_name}} | 汇报人 | 文本 | 必填 | 审核员小王 / 张组长 |
| {{raw_work_logs}} | 原始工作记录 | 长文本 | 必填 | 工单、会议、策略、异常、待办完整碎片 |
| {{focus_business}} | 本周核心业务 | 数组 | 必填 | ["短视频虚假广告治理","群聊涉赌拦截优化"] |
| {{last_week_risk}} | 上周遗留风险 | 文本 | 选填 | 无 / API审核超时、模型误判率偏高 |
| {{output_limit}} | 单条事项字数 | 数字 | 选填 | 默认30 |
| {{week_type}} | 周类型标记 | 文本 | 选填 | 正常周 / 故障多发周 / 假期交接周 |

## 强制分步执行规则（不可修改）
严格执行4步，每一步输出分段结果，禁止跳步、合并信息：
步骤1：对{{raw_work_logs}}自动切分为3独立模块：
① 审核工单数据；② 会议纪要+策略更新；③ 异常事件+个人待办
> 补充约束：若模块③存在≥2起独立故障，必须逐条拆分：发生时间、故障现象、业务影响、临时处置、长期整改方案，禁止简写合并。
步骤2：逐段结构化提取全部量化数据、违规类型、策略变更、待办进度。
步骤3：结合{{last_week_risk}}识别跨周持续风险；根据{{week_type}}调整内容权重：
- 故障多发周：放大异常事件、技术风险、整改计划篇幅
- 假期交接周：重点展示人员排班、行政交接、假期应急事项
- 正常周：侧重工单数据、内容识别优化、常规迭代
步骤4：整合全部提取信息，按固定格式输出完整周报，所有数据必须来源于原始日志，不得编造。

## 固定输出格式（强制分类）
# 内容安全审核团队周报【{{report_cycle}}】
汇报人：{{reporter_name}}
周类型：{{week_type}}
📌 本周概要（≤100字，贴合{{week_type}}业务特征）
📊 关键数据
总工单总量、日均量、通过/拒绝/复审数量及占比、违规类型分布、系统故障量化影响
✅ 本周完成（工单/策略/会议/异常处置四类，每条≤{{output_limit}}字）
🔄 进行中事项（标注完成进度、截止时间）
⚠️ 风险与问题（分三类展示：线上技术风险、内容识别风险、人员/业务风险；延续{{last_week_risk}}跟踪）
📅 下周计划（逐条匹配本周风险与{{focus_business}}核心业务）
