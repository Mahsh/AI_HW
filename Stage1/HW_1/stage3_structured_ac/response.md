# 超级SIM卡数字身份功能需求说明书
## 1. 文档概述
### 1.1 文档目的
本需求说明书旨在明确超级SIM卡数字身份功能的产品需求，涵盖功能范围、核心能力、非功能要求等内容，为产品设计、开发、测试及交付提供统一的指导依据。
### 1.2 文档范围
本说明书适用于超级SIM卡数字身份功能第一版的研发与交付，二期功能（电子签名）及海外适配不在本次范围之内。
### 1.3 参考依据
超级SIM卡数字身份功能需求沟通记录（上周五下午产品规划讨论会）。
### 1.4 术语与定义
| 术语 | 定义 |
|------|------|
| SIM卡安全芯片 | 超级SIM卡内置的具备加密、存储能力的硬件芯片，为本功能提供安全基础 |
| NFC标准门禁读头 | 支持NFC协议的门禁读取设备，可识别超级SIM卡内置的身份凭证 |
| 核心凭证 | 存储于SIM卡芯片的身份认证关键信息（如证书、密钥） |
| 完整身份信息 | 除核心凭证外的详细身份数据（如员工信息、权限信息等） |
| 等保 | 网络安全等级保护，是针对信息系统安全的合规要求 |

## 2. 业务背景与目标
### 2.1 业务背景
当前数字身份领域市场热度高，支付宝、微信等平台均在布局相关功能；超级SIM卡内置安全芯片，具备天然的安全优势，相较于基于手机号的移动认证，安全等级更高，具备差异化竞争力。政企客户对身份认证的便捷性、安全性、离线可用性有明确需求，是本次功能的核心目标用户。
### 2.2 核心目标
- 依托超级SIM卡安全芯片，快速上线数字身份核心功能，抢占政企客户市场；
- 满足政企客户办公门禁、APP一键登录的核心场景需求，提升用户体验；
- 建立企业端管理能力，支撑政企客户批量运维需求；
- 保障功能合规性与安全性，符合相关法规及等保要求。

## 3. 功能需求
### 3.1 核心功能模块
#### 3.1.1 政企办公门禁认证
- **功能描述**：用户可通过超级SIM卡刷NFC标准门禁读头完成门禁认证，支持离线场景使用；
- **关键需求**：
  - 兼容市面上70%以上的NFC标准门禁读头，覆盖主流厂商（如海康、大华、中控等），完成NFC不同协议变种的兼容性测试；
  - 离线场景下，预加载身份凭证到本地缓存，支持门禁认证，缓存凭证需设置合理的过期时间（平衡用户体验与安全风险）；
  - 门禁认证结果实时反馈（如门禁设备端的成功/失败提示）。

#### 3.1.2 APP一键登录
- **功能描述**：用户通过超级SIM卡芯片完成APP身份认证，实现一键登录，无需输入密码；
- **关键需求**：
  - 与现有基于手机号的移动认证产品区分，核心依托SIM卡芯片完成认证，提升安全等级；
  - 登录流程简洁，单次认证操作步骤不超过2步；
  - 支持与政企客户自有APP的对接适配。

#### 3.1.3 身份信息存储与管理
- **功能描述**：合理规划身份信息存储方式，保障数据安全与使用便捷性；
- **关键需求**：
  - 核心凭证（证书、密钥等）存储于SIM卡安全芯片（适配128KB可用存储空间）；
  - 完整身份信息存储于服务端，使用时实时调取；
  - 支持社保卡、医保卡等电子证照信息存储的预留扩展（需提前完成合规评估）。

#### 3.1.4 用户端管理功能
- **功能描述**：通过H5页面/小程序为用户提供数字身份凭证管理能力；
- **关键需求**：
  - 支持查看已绑定的身份凭证列表（如门禁凭证、登录凭证）；
  - 支持查看凭证过期时间，提供凭证更新提醒及更新操作入口；
  - 界面简洁易懂，适配普通用户的操作习惯，配套用户教育引导内容（如功能使用说明）。

#### 3.1.5 企业端管理后台
- **功能描述**：为企业管理员提供批量运维能力，优先级高于普通用户端；
- **关键需求**：
  - 员工权限管理：批量开通/撤销员工的SIM卡数字身份权限（门禁、APP登录）；
  - 日志管理：查看员工数字身份使用记录（如门禁刷卡时间、APP登录时间）；
  - 权限配置：可针对不同员工配置不同的门禁权限、APP登录权限。

### 3.2 一期功能边界
- 不支持电子签名功能（规划为二期）；
- 不支持海外适配（需单独评估，暂不纳入）；
- 门禁对接仅支持NFC标准读头，非NFC标准门禁按需后续接入。

## 4. 非功能需求
### 4.1 安全性要求
- 核心凭证存储于SIM卡芯片，具备防篡改、防窃取能力；
- 系统需通过网络安全等级保护认证，若涉及金融级场景，需满足对应安全认证要求；
- 离线缓存凭证设置合理过期时间，降低安全风险；
- 符合《个人信息保护法》《数据安全法》等法规要求，存储/使用身份信息前完成法务合规评估。

### 4.2 性能要求
- 门禁认证响应时间≤1秒（在线场景），离线场景认证响应时间≤2秒；
- APP一键登录认证响应时间≤3秒；
- 服务端支持至少1000个企业租户、10万级用户的并发访问。

### 4.3 兼容性要求
- SIM卡端：适配国内主流型号的超级SIM卡（128KB可用存储空间）；
- 门禁设备：兼容70%以上的NFC标准门禁读头，支持NFC不同协议变种；
- 用户端：H5页面适配主流浏览器（Chrome、微信内置浏览器、Safari等），小程序适配微信/支付宝平台。

### 4.4 可用性要求
- 用户端操作界面简洁，核心功能操作路径不超过3步；
- 企业管理后台提供操作指引，支持管理员快速上手；
- 提供故障提示机制（如门禁认证失败时明确提示原因：凭证过期、网络异常、权限不足等）。

### 4.5 交付时间要求
第一版功能需在下个月底前完成上线。

## 5. 合规与法务要求
- 接入社保卡、医保卡等政府电子证照信息前，完成法务咨询，确保符合相关合规要求；
- 个人身份信息的收集、存储、使用需严格遵循《个人信息保护法》《数据安全法》；
- 系统安全合规需满足等保及金融级（如有）认证要求。

## 6. 验收标准
### 6.1 功能验收
- 门禁认证：在线/离线场景均可正常认证，兼容目标NFC门禁设备；
- APP一键登录：完成与至少1个政企客户APP的对接，实现一键登录；
- 存储管理：核心凭证存储于SIM卡，完整信息存储于服务端，调取正常；
- 用户端：可查看/更新凭证，引导内容清晰；
- 企业端：可批量管理权限、查看使用日志。

### 6.2 非功能验收
- 安全性：通过等保认证，核心凭证无泄露风险，合规审查无重大问题；
- 性能：各项响应时间满足要求，并发访问无明显卡顿；
- 兼容性：完成目标设备/平台的兼容性测试，无重大兼容性问题。
